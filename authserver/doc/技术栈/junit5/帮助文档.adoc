
== JUnit Jupiter 支持以下注释来配置测试和扩展框架。

:junit-api-url: https://junit.org/junit5/docs/current/api/org.junit.jupiter.api/org/junit/jupiter/api/package-summary.html
除非另有说明，所有核心注解均位于模块 {junit-api-url}[org.junit.jupiter.api] 中的包中junit-jupiter-api。

:ParameterizedTest-url: https://junit.org/junit5/docs/current/user-guide/#writing-tests-parameterized-tests

:RepeatedTest-url: https://junit.org/junit5/docs/current/user-guide/#writing-tests-repeated-tests
:TestFactory-url: https://junit.org/junit5/docs/current/user-guide/#writing-tests-dynamic-tests

:TestTemplate-url: https://junit.org/junit5/docs/current/user-guide/#writing-tests-test-templates
:TestTemplate-provide-utl: https://junit.org/junit5/docs/current/user-guide/#extensions-test-templates

:TestClassOrder-url: https://junit.org/junit5/docs/current/user-guide/#writing-tests-test-execution-order-classes

:TestMethodOrder-utl: https://junit.org/junit5/docs/current/user-guide/#writing-tests-test-execution-order-methods
:TestInstance-url: https://junit.org/junit5/docs/current/user-guide/#writing-tests-test-instance-lifecycle
:DisplayName-url: https://junit.org/junit5/docs/current/user-guide/#writing-tests-display-names
:DisplayNameGeneration-url: https://junit.org/junit5/docs/current/user-guide/#writing-tests-display-name-generator

:Nested-url: https://junit.org/junit5/docs/current/user-guide/#writing-tests-nested

:Tag-url: https://junit.org/junit5/docs/current/user-guide/#writing-tests-tagging-and-filtering

:Disabled-url: https://junit.org/junit5/docs/current/user-guide/#writing-tests-disabling

:ExtendWith-url: https://junit.org/junit5/docs/current/user-guide/#extensions-registration-declarative

:RegisterExtension-url: https://junit.org/junit5/docs/current/user-guide/#extensions-registration-programmatic

:TempDir: https://junit.org/junit5/docs/current/user-guide/#writing-tests-built-in-extensions-TempDirectory

|===
|注解	|描述
|@Test
|表示方法是测试方法。与 JUnit 4 的@Test注释不同，此注释不声明任何属性，因为 JUnit Jupiter 中的测试扩展基于其自己的专用注释进行操作。此类方法将被继承，除非它们被重写。

|@ParameterizedTest
|表示方法是 {ParameterizedTest-url}[参数化测试]。此类方法将被继承，除非它们被重写。

|@RepeatedTest
|表示方法是 {RepeatedTest-url}[重复测试]的测试模板。此类方法将被继承，除非它们被重写。

|@TestFactory
|表示方法是 {TestFactory-url}[动态测试]的测试工厂。此类方法将被继承，除非它们被重写。

|@TestTemplate
|表示方法是 {TestTemplate-url}[测试用例的模板   ]，设计为根据注册 {TestTemplate-provide-utl}[提供者]返回的调用上下文的数量多次调用。此类方法将被继承，除非它们被重写。

|@TestClassOrder
|用于配置带注释的测试类中测试类的 {TestClassOrder-url}[测试类执行顺序]@Nested。此类注释是继承的。

|@TestMethodOrder
|用于配置被注解的测试类的 {TestMethodOrder-utl}[测试方法执行顺序]；类似于 JUnit 4 的@FixMethodOrder. 此类注释是继承的。

|@TestInstance
|用于配置带注释的测试类的 {TestInstance-url}[测试实例生命周期]。此类注释是继承的。

|@DisplayName
|声明测试类或测试方法的自定义 {DisplayName}[显示名称]。此类注释不会被继承。

|@DisplayNameGeneration
|为测试类声明一个自定义 {DisplayNameGeneration}[显示名称生成器]。此类注释是继承的。

|@BeforeEach
|表示被注解的方法应该在当前类中的每个, , , 或方法之前执行； 类似于 JUnit 4 的. 此类方法是继承的，除非它们被覆盖或取代（即仅基于签名进行替换，而不考虑 Java 的可见性规则）。 @Test@RepeatedTest@ParameterizedTest@TestFactory@Before

|@AfterEach
|表示被注解的方法应该在当前类中的每个、、、 或方法之后执行； 类似于 JUnit 4 的. 此类方法是继承的，除非它们被覆盖或取代（即仅基于签名进行替换，而不考虑 Java 的可见性规则）。 @Test@RepeatedTest@ParameterizedTest@TestFactory@After

|@BeforeAll
|表示被注解的方法应该在当前类中的所有、、、方法之前执行； 类似于 JUnit 4 的. 此类方法是继承的，除非它们被隐藏、覆盖或取代（即，仅根据签名进行替换，而不考虑 Java 的可见性规则），而且必须如此，除非使用“每类” {TestInstance-url}[测试实例生命周期]。 @Test@RepeatedTest@ParameterizedTest@TestFactory@BeforeClassstatic

|@AfterAll
|表示被注解的方法应该在当前类中的所有、、、方法之后执行； 类似于 JUnit 4 的. 此类方法是继承的，除非它们被隐藏、覆盖或取代（即，仅根据签名进行替换，而不考虑 Java 的可见性规则），而且必须如此，除非使用“每类” {TestInstance-url}[测试实例生命周期]。 @Test@RepeatedTest@ParameterizedTest@TestFactory@AfterClassstatic

|@Nested
|表示被注解的类是一个非静态嵌套测试类。在 Java 8 到 Java 15 上，@BeforeAll除非使用“每类” {TestInstance-url}[测试实例生命周期]@AfterAll，否则不能直接在测试类中使用方法。从 Java 16 开始，可以使用任一测试实例生命周期模式在测试类中声明方法。此类注释不会被继承。@Nested@BeforeAll@AfterAllstatic@Nested

|@Tag
|用于在类或方法级别声明用于 {Tag-url}[过滤测试的标签]；类似于 TestNG 中的测试组或 JUnit 4 中的类别。此类注释在类级别继承，但不在方法级别继承。

|@Disabled
|用于 {Disabled-url}[禁用]测试类或测试方法；类似于 JUnit 4 的@Ignore. 此类注释不会被继承。

|@Timeout
|用于在测试、测试工厂、测试模板或生命周期方法的执行超过给定持续时间时失败。此类注释是继承的。

|@ExtendWith
|用于以 {ExtendWith-url}[声明方式注册扩展]。此类注释是继承的。

|@RegisterExtension
|用于通过字段 {RegisterExtension-url}[以编程方式注册扩展]。这些字段是继承的，除非它们被遮蔽。

|@TempDir
|用于在生命周期方法或测试方法中通过字段注入或参数注入提供 {TempDir}[临时目录]；位于org.junit.jupiter.api.io包装内。
|===

TIP: 某些注释目前可能处于实验阶段。有关详细信息，请参阅实验 API中的表 。
